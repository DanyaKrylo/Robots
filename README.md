# Robots — учебный проект с графическим интерфейсом на Java Swing

Проект демонстрирует поэтапную доработку приложения с интерфейсом на `javax.swing`.
Приложение управляет виртуальным «роботом» на плоскости, имеет окно лога, меню,
поддерживает локализацию и сохранение/восстановление состояния окон.

## Возможности приложения

- Окно игрового поля с роботом, который двигается к цели (клик мышью).
- Окно протокола работы с ограничением длины очереди сообщений.
- Меню с переключением темы оформления (Look & Feel), языка интерфейса и тестовыми командами.
- Подтверждение закрытия главного окна и внутренних окон.
- Локализация на русский и английский языки (по умолчанию — системный язык, смена через меню).
- Профили состояния: сохранение расположения и состояния окон при выходе, выбор профиля при запуске.

## Сборка и запуск

Проект собирается с помощью **Apache Maven**. Требуется JDK 11 или выше.

### Команды

```bash
# Компиляция исходного кода
mvn compile

# Запуск приложения
mvn exec:java

# Запуск тестов
mvn test

# Очистка собранных файлов
mvn clean
```

## Выполненные задачи (по этапам)

### 1. Настройка сборки с Apache Maven

- Добавлен `pom.xml` с зависимостями: `gson` (для сохранения профилей), `junit-jupiter` (для тестов).
- Настроены плагины `exec-maven-plugin` для запуска приложения и `maven-surefire-plugin` для тестов.
- Исходный код и ресурсы перемещены в стандартные каталоги Maven.

### 2. Исправление багов в `LogWindowSource`

- **Утечка памяти:** слушатели теперь отписываются при закрытии окна лога (метод `unregisterListener`).
- **Неограниченный рост лога:** очередь сообщений теперь хранится в `Deque<LogEntry>` с ограничением `m_iQueueLength`; старые записи удаляются при добавлении новых.

### 3. Исправление «убегания» робота за границу окна

- В методе `GameVisualizer.moveRobot()` новые координаты робота ограничиваются размерами панели (`getBounds()`), чтобы робот всегда оставался в пределах видимой области.

### 4. Подтверждение закрытия окон

- Создан вспомогательный класс `CloseDialogHelper` с методом `confirmClose()` для вызова диалога подтверждения с локализованными сообщениями.
- Подтверждение запрашивается при закрытии:
  - главного окна (крестик или Alt+F4),
  - внутренних окон «Игровое поле» и «Протокол работы» (крестик на вкладке),
  - через пункт меню «Выйти».

### 5. Пункт меню «Выйти»

- В строке меню добавлено меню «Файл» с пунктом «Выйти». Обработчик аналогичен закрытию главного окна — показывает диалог подтверждения и завершает приложение.

### 6. Локализация приложения

- Реализован синглтон `LocaleManager`, загружающий `ResourceBundle` (`messages.properties` и `messages_en.properties`).
- Все элементы интерфейса (меню, заголовки окон, диалоги подтверждения) используют строки из ресурсов.
- По умолчанию выбирается язык операционной системы (`Locale.getDefault()`).
- Через меню «Язык» можно переключиться на «Русский» или «English», после чего все строки динамически обновляются.

### 7. Сохранение и восстановление профиля состояния окон

- Профиль содержит:
  - локаль (язык),
  - положение и размер главного окна,
  - расширенное состояние главного окна (развёрнуто/свёрнуто),
  - для каждого внутреннего окна: тип, координаты, размеры, свернутость и закрытость.
- При выходе из приложения через «Выйти» или закрытие главного окна текущий профиль автоматически сохраняется в JSON-файл в папке `profiles/`.
- При следующем запуске приложение сканирует папку `profiles`, и если там есть файлы, выводит диалог со списком доступных профилей. Пользователь может выбрать один из них для восстановления или отказаться (кнопка «Не восстанавливать»).
- Выбранный профиль восстанавливает язык, положение и состояние всех окон.
- Поддерживается несколько профилей (каждый сохраняется с временной меткой), что позволяет вернуться к любому ранее сохранённому состоянию.

### 8. Тестирование

- Написан тестовый класс `LogWindowSourceTest` с использованием JUnit 5.
- Тесты проверяют:
  - ограничение размера очереди сообщений (старые записи удаляются),
  - корректность вызова слушателей при добавлении сообщения,
  - правильность отписки слушателей.

## Зависимости (указываются в `pom.xml`)

- **Gson** (2.10.1) — сериализация профилей в JSON.
- **JUnit Jupiter** (5.10.1) — модульное тестирование (только для тестов, scope `test`).

## Примечания

- Для работы приложения файлы локализации (`messages.properties` и `messages_en.properties`) должны находиться в classpath (в папке `resources`).
- При необходимости смены системной локали для тестирования можно передать параметры JVM:  
  `mvn exec:java -Dexec.args="-Duser.language=en -Duser.country=US"`
